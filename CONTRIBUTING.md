# Guia de Contribuição

Obrigado por seu interesse em contribuir com a Tech Fernandes LTDA!

## Código de Conduta

Este projeto e toda a organização seguem nosso [Código de Conduta](CODE_OF_CONDUCT.md). Ao participar, espera-se que você mantenha este código.

## Como Contribuir

1. **Issues**: Utilize as issues para reportar bugs ou sugerir novas funcionalidades. Verifique se já não existe uma issue similar.
2. **Pull Requests**:
   - Faça um fork do repositório
   - Crie um branch para sua feature (`git checkout -b feat/minha-feature`)
   - Commit suas mudanças (`git commit -m 'feat: adiciona minha-feature'`)
   - Push para o branch (`git push origin feat/minha-feature`)
   - Abra um Pull Request

## Padrões de Commit

Seguimos a especificação [Conventional Commits](https://www.conventionalcommits.org/pt-br/).

- `feat`: Nova funcionalidade
- `fix`: Correção de bug
- `docs`: Documentação
- `style`: Formatação, falta de ponto e vírgula, etc.
- `refactor`: Refatoração de código
- `test`: Adição de testes
- `chore`: Atualização de build, ferramentas, etc.

## Revisão de Código

Todos os PRs passam por revisão. Seja paciente e respeitoso durante o processo.
